================================================================================
TRANSLATION RULES — MANDATORY — NEVER VIOLATE ANY RULE
================================================================================
SIGNAL RULES:
  R1. Use ONLY signal names from AVAILABLE SIGNALS. Zero exceptions.
  R2. Signal widths must match — check the width column before using a signal.
  R3. Never concatenate or slice signals unless the NS31A assertion explicitly
      requires a sub-field that maps to a known Ibex bit range.
  R4. For enum-typed signals: use symbolic constant names from PACKAGE TYPE
      DEFINITIONS above. Never hardcode raw bit values (e.g., 2'b11).
  R5. For struct-typed signals: use dot notation with exact field names from
      PACKAGE TYPE DEFINITIONS above. Never invent or guess field names.

SVA SYNTAX RULES (COMBINATIONAL):
  R6. FORBIDDEN: @(posedge ...), ##N, $past(), $rose(), $fell(),
      disable iff, always_ff, posedge, negedge, clocking blocks.
      Any use of these is a compilation error for this module type.
  R7. Use concurrent assertion syntax WITHOUT a clocking event:
        property PROP_NAME;
          <boolean expression> |-> <boolean expression>;
        endproperty
        assert property (PROP_NAME);
      OR use simple assign-level immediate assertions:
        always_comb begin
          PROP_NAME: assert (<boolean expression>);
        end
  R8. For pure combinational implication (if A then B must hold now):
        property PROP_NAME;
          <condition> |-> <consequence>;
        endproperty
  R9. Every property must have both a named property block AND a named assert:
        property PROP_NAME; <antecedent> |-> <consequent>; endproperty
        assert property (PROP_NAME);
  R10. Property names: {{MODULE_SHORT}}_SEC_<N> where N is a sequential integer.
  R11. Antecedent must be reachable in normal (non-Trojan) operation.
       If the antecedent can never be true, rewrite it.
  R12. No $display, $error, or simulation-only constructs inside properties.

BIND FILE RULES:
  R13. Port list of the assertion module must match the DUT port list exactly.
       Use (input/output + width + name) for each port.
  R14. NO clock or reset in the port list (combinational module has neither).
  R15. Bind statement goes OUTSIDE and AFTER the assertion module:
         bind {{MODULE_NAME}} {{MODULE_NAME}}_assertions u_{{MODULE_SHORT}}_assert (.*);
  R16. The (.*) wildcard connects all ports by name — valid only if port names
       in the assertion module exactly match DUT port names.

UNTRANSLATABLE RULES:
  R17. If a NS31A signal has no Ibex equivalent: mark status "flagged" in the
       JSON log and skip that assertion entirely. Do not guess or approximate.
  R18. If an assertion references MULTIPLE untranslatable signals: one "flagged"
       entry covers all of them — do not create a partial assertion.
